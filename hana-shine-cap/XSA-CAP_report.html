1. About

This section describes the Hana Cloud Compatible DB Artifacts migrated from XS Advanced, as well as any limitations or differences in behavior, where applicable.

2.Migration Status

HANA native artifacts such as 'hdbtable', 'hdbcalculationviews', 'hdbview', and 'hdbfunctions' will not be read by CDS. To make these objects recognizable to CDS, respective proxy files are created with certain limitations. For more information on this,please check the Hana Cloud Documentation.

These Proxy entities are created under their respective folders with the tag Proxy_*.cds

In calculation Views unsupported data types date(), day(), time() and counter datatype ( Integer ) are converted into respective supported data types daydate(), daytime(), secondtime() and counter datatype ( BIGINT ) .

While transitioning to CDS, the migration process involves renaming tables. Consequently, all other HANA artifacts that use these tables or views are also updated.

Table type definitions are not supported in Hana Cloud. Hence, during the migration, respective .hdbtabletype definitions are created.

The following features are currently out of scope:

  • Converting xsodata into cap service definition
  • Converting “xsjs","xsjslib" into cap nodejs
  • Creating proxy cds for cross container schema
  • Following Artifacts are not currently supported '.hdbreptask', '.hdbvirtualtable', '.hdbflowgraph'

When creating a Level Hierarchy in a calculation view for SAP HANA Cloud, the Node Style option defaults to Name Path; Level Name or Name Only are not allowed.

4.Post Migration Steps

Migration of DB Artifacts from XS to CAP is complete. However, please verify these steps post migration:

  • SQL syntax changes in procedure, ex: UPDATE FROM has to be changed into MERGE INTO, TRUNCATE statement with a DELETE FROM statement

  • Flowgraph, Reptask and Replication artifacts Changes are not covered.

  • Unsupported types and functions in calculation view ex: "CE_FUNCTION", “CACHE" etc, please check the Hana Cloud documentation

  • Verify the cross container artifacts before deployment
